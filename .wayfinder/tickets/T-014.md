# T-014: Notification System Specification

`wayfinder:task` | Parent: map.md

## Question

Produce the specification for the notification system: in-app notifications, push notifications (Android/iOS), and the pluggable notification channel interface for future channels (email, Slack, Telegram).

## Scope

### In-App Notifications
1. **Notification model** — title_code, body_code, payload (JSONB with dispatch_id, campaign_id, etc.), read_at
2. **Notification triggers** — when are notifications created? Dispatch pending, dispatch late, dispatch missed, campaign completed, import finished, team member added
3. **Notification UI** — bell icon with unread count, dropdown list, click to navigate to context
4. **Mark as read** — single + mark all, auto-mark when navigating to context
5. **Persistence** — notifications stored in DB, paginated query

### Push Notifications
1. **Platform support** — Android (FCM), iOS (APNs)
2. **Device registration** — register device token per user, handle token refresh
3. **Push payload** — title, body, data payload (for deep linking), sound (configurable), badge count
4. **Delivery** — via Supabase Edge Function or dedicated push service (OneSignal, Firebase)
5. **Channel interface** — `PushNotificationChannel` implements `NotificationChannel`

### Pluggable Channel Interface
```typescript
interface NotificationChannel {
  send(notification: Notification): Promise<void>;
  getDeliveryStatus(notificationId: string): Promise<DeliveryStatus>;
}
```
Future implementations: `EmailChannel`, `SlackChannel`, `TelegramChannel` — spec the interface only, not implementations.

### Notification Preferences
1. Per-user: which channels enabled, quiet hours
2. Per-notification-type: which types go to which channels
