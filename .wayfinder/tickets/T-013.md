# T-013: Import Pipeline Specification

`wayfinder:task` | Parent: map.md

## Question

Produce the complete specification for the product import pipeline: Raw Import → Extract → Candidate → Product. Covers CSV import with column mapping, WhatsApp ZIP import with media-message pairing, and the candidate review/approval workflow.

## Scope

### CSV Import
1. **Upload** — multipart form data, file size limit, accepted formats (CSV, TSV)
2. **Parsing** — detect delimiter, encoding, header row
3. **Column Mapping** — UI flow: preview first 5 rows, user maps columns to Product fields (name, selling price, cost price, promotional price, currency, description, stock, product code)
4. **Import Formulas** — suggested transformations per column (multiply, divide, regex extract, remove symbol, split), preview before accept
5. **Price Classification** — for each price column, user selects: selling price, cost price, or promotional price
6. **Batch import** — all rows imported as Candidates (not Products) with batch_id

### WhatsApp ZIP Import
1. **Upload** — ZIP file, extract contents, identify media files + text files
2. **Media-message pairing** — algorithm to pair each message with its corresponding media file(s) based on timestamps or filename patterns in WhatsApp exports
3. **Extract generation** — each paired message+media becomes an Extract
4. **Chronological display** — extracts shown in original WhatsApp conversation order
5. **Filtering UI** — user marks irrelevant extracts as discarded, keeps relevant ones

### Shopify / Website Import
1. **Integration pattern** — fetch products via API (Shopify Admin API) or scrape (website)
2. **Field mapping** — Shopify/website fields → Product fields
3. **Media handling** — download and re-upload to R2, generate presigned URLs

### Candidate Review
1. **Review queue** — list of candidates pending approval, filterable by batch, source
2. **Prefill** — candidates pre-filled with Parsing Rule extraction or AI extraction results
3. **Field highlighting** — probabilistically identified fields highlighted with confidence indicator (green/yellow/red)
4. **Manual correction** — user adjusts any field inline
5. **AI toggle** — per-candidate AI extraction toggle (cost-tracked)
6. **Media preview** — images displayed inline in review UI
7. **Approve/Reject** — approve → creates Product (with variants if multiple media), reject → status updated with reason (logged in Action Log)
8. **Bulk approve** — approve all confidently-parsed candidates in one action

### Parsing Rules (per Supplier)
1. **Rule creation** — regex or pattern per field (name pattern, price pattern, currency pattern)
2. **Rule testing** — test rule against sample extract text, see what it extracts
3. **Rule priority** — multiple rules per field, applied in priority order, first match wins
4. **Rule learning** — Action Log data feeds AI to suggest new rules (future, not initial spec)
