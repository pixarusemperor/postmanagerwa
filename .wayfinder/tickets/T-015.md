# T-015: Integration & Migration Specification

`wayfinder:task` | Parent: map.md

## Question

Produce the specification for how PostManager integrates with external systems and the migration path from existing WassFlow. This is the "pluggability" spec ‚Äî how other systems connect.

## Scope

### Schema Handshake: pm ‚Üî wa
1. **`pm.enqueue_dispatch()` Postgres function** ‚Äî how PostManager inserts a dispatch into `wa.send_queue` for automated sending
2. **Return channel** ‚Äî how wa reports dispatch status back to pm (direct DB update, webhook, or shared function)
3. **WA Poster ‚Üî WA Instance mapping** ‚Äî `pm.wa_posters` and `wa.wa_instances` refer to the same WhatsApp session but in different schemas ‚Äî how are they kept in sync? Shared `session_id`? Single source of truth?
4. **Cross-schema queries** ‚Äî can pm queries read from wa tables and vice versa? Or strictly separated with explicit handshake points?

### External System Integration
1. **Export API consumer contract** ‚Äî any external system can call `GET /api/pm/campaigns/:id/export` and receive the full dispatch list as JSON. Spec the exact JSON schema.
2. **Status webhook contract** ‚Äî external system calls `POST /api/pm/webhooks/dispatch-status` to report back. Spec the exact payload.
3. **API key auth for external systems** ‚Äî how do external systems authenticate? Organization-level API keys? Per-campaign webhook URLs?
4. **Retry and idempotency** ‚Äî how does the webhook handle duplicate status reports? Idempotency key?

### WassFlow Migration
1. **Data migration approach** ‚Äî WassFlow's Supabase tables ‚Üí new wa schema tables. Mapping document: `wf_config` ‚Üí `wa_instances`, `wf_sequences` ‚Üí `wa_sequences`, etc.
2. **Downtime strategy** ‚Äî can migration happen with zero downtime? Blue-green? Read-only window?
3. **Backward compatibility** ‚Äî does PostManager need to support WassFlow's old API for a transition period?
4. **Feature parity** ‚Äî what WassFlow features are NOT absorbed? What gets dropped?
5. **URL migration** ‚Äî WassFlow at `wassflow.orizongroup.online` ‚Üí PostManager at new subdomain. Redirect strategy?
6. **Session migration** ‚Äî existing WhatsApp sessions (QR-paired) must survive migration. How?

### ERP / Inventory / External DB Integration (Future)
1. **Data source interface** ‚Äî abstract interface for plugging in external product databases
2. **Sync strategy** ‚Äî pull vs push, real-time vs batch, conflict resolution
3. **Field mapping** ‚Äî external field ‚Üí Product field, configurable per data source
