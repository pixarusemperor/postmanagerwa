# T-007: Frontend Architecture & Route Specification

`wayfinder:task` | Parent: map.md

## Question

Produce the complete frontend specification: Next.js App Router route structure, page-level component tree, layout hierarchy, state management strategy, and key UI patterns (drag-and-drop, responsive design, clipboard paste). This is the blueprint for the UI implementation.

## Scope

### Route Structure
Every page route with its URL, layout, auth requirements, and main components:

```
/(auth)/                     ← public auth pages (login, signup, forgot password)
/(dashboard)/                ← authenticated layout shell
  /organizations/            ← org switcher + settings
  /products/                 ← product list with search/filter/category
  /products/[id]/            ← single product detail + variants
  /products/import/          ← import wizard (CSV, ZIP, manual)
  /products/import/[id]/     ← import review (extracts → candidates)
  /categories/               ← category tree management
  /suppliers/                ← supplier list
  /suppliers/[id]/           ← supplier detail + parsing rules
  /templates/                ← caption + prefilled message templates
  /campaigns/                ← campaign list
  /campaigns/new/            ← campaign creation wizard
  /campaigns/[id]/           ← campaign detail + timeline + dispatches
  /campaigns/[id]/edit/      ← edit campaign (posts, targets, config)
  /targets/                  ← target lists management
  /wa-posters/               ← WhatsApp API connections
  /contact-numbers/           ← contact numbers management
  /poster/                   ← Human Poster dashboard (today's dispatches)
  /poster/performance/       ← performance stats
  /discounts/                ← discount rules
  /ai-settings/              ← AI configuration
  /notifications/            ← notification inbox
  /action-log/               ← action log viewer
  /settings/                 ← org settings, language, working hours
  /wa/                       ← WhatsApp automation (separate section)
  /wa/instances/             ← WhatsApp session management
  /wa/sequences/             ← automation sequences
  /wa/triggers/              ← keyword triggers
  /wa/inbox/                 ← WhatsApp message inbox
  /wa/campaigns/             ← WhatsApp bulk campaigns
  /wa/group-lists/           ← group lists
  /wa/send-queue/            ← send queue monitor
```

### Component Tree
Major reusable components with their props interfaces:
- **DashboardLayout** — sidebar, org switcher, user menu, notification bell
- **ProductCard** / **ProductGrid** — thumbnail grid with drag selection
- **CampaignTimeline** — drag-and-drop post ordering (mouse + touch)
- **DispatchQueue** — today's pending dispatches with "Mark as Done"
- **TemplateEditor** — rich text with variable autocomplete and formatting toolbar
- **ColumnMapper** — CSV column → field mapping wizard
- **ImportReviewPipeline** — extract list with media preview, filter/reject, promote
- **CategoryTree** — hierarchical category browser with drag reorganization
- **CampaignPreviewPanel** — timeline preview before campaign creation
- **AntiDetectionConfigPanel** — constraint configuration form
- **WAInstanceCard** — instance status with QR pairing
- **SequenceFlowCanvas** — visual sequence builder
- **ActionLogViewer** — filterable, paginated log table

### State Management
- Server state: Supabase realtime subscriptions or React Query for lists (products, campaigns, dispatches)
- Client state: Zustand or React Context for dashboard state, selected org, language
- Form state: React Hook Form + Zod validation
- Drag-and-drop: @dnd-kit/core for post ordering and category tree

### Key UX Patterns
- **Drag-and-drop** — mouse + touch, visual feedback, snap positions
- **Responsive** — mobile-first breakpoints for Poster mobile use, desktop for campaign management
- **Clipboard paste** — paste image from clipboard on product creation
- **Bulk selection** — checkboxes + shift-click + drag selection on product grids
- **Inline editing** — edit product fields inline in table view
- **Toast notifications** — for async action feedback (dispatch sent, import completed)

### Language / i18n
- Next.js i18n routing or middleware-based locale detection
- All UI strings via translation keys (never hardcoded text)
- Organization default + user override
- RTL support for Arabic
