# T-001: Database Schema — PostManager Tables (pm.*)

`wayfinder:task` | Parent: map.md

## Question

Produce the complete DDL for the `pm` Postgres schema. Every PostManager table, column, constraint, index, and extension setup. This is the authoritative schema document that all other specs reference.

## Scope

Must cover these tables and their columns:

- **organizations** — id, name, slug, plan, settings (JSONB), created_at
- **organization_members** — organization_id (FK), user_id (FK → auth.users), role (enum)
- **products** — id, organization_id, code (unique per org), name, selling_price, cost_price, promotional_price, currency, discounted_price, stock_count, description, created_at, updated_at
- **product_variants** — id, product_id (FK), media_url, caption, display_order
- **product_categories** — id, organization_id, name, path (ltree), parent_id (self-referencing FK)
- **product_category_members** — product_id, category_id (M2M join)
- **product_suppliers** — product_id, supplier_id (M2M join)
- **suppliers** — id, organization_id, name, contact_info (JSONB), created_at
- **parsing_rules** — id, supplier_id (FK), field_name, rule_pattern (regex), priority, created_at
- **raw_imports** — id, organization_id, source_type (enum: csv, whatsapp_zip, shopify, website, manual), file_url, status (enum: pending, processing, completed, failed), batch_id, created_at
- **extracts** — id, raw_import_id (FK), message_text, media_url, sender_number, timestamp, status (enum: pending, filtered, promoted)
- **candidates** — id, extract_id (FK), product_id (FK, nullable until promoted), suggested_name, suggested_price, suggested_fields (JSONB), status (enum: pending_approval, approved, rejected), reviewed_by (FK → auth.users), reviewed_at
- **campaigns** — id, organization_id, name, status (enum: draft, scheduled, running, paused, completed, cancelled), post_mode (enum: manual, automated, export), anti_detection_config_id (FK), created_by (FK → auth.users), created_at, updated_at
- **posts** — id, campaign_id (FK), position (int), type (enum: text_only, media_text, grouped_media_text_before, grouped_media_text_after, audio, document), snapshot (JSONB — frozen product data at creation), caption_override (text, nullable), created_at
- **media_groups** — id, post_id (FK, nullable — one post can have one media group)
- **media_group_items** — id, media_group_id (FK), media_id (FK), position (int), type (enum: image, video)
- **media** — id, organization_id, storage_path, bucket_name, mime_type, file_size, product_id (FK, nullable), variant_id (FK, nullable), extract_id (FK, nullable), created_at
- **target_lists** — id, organization_id, name, description, created_at
- **targets** — id, target_list_id (FK), type (enum: group, individual), jid (WhatsApp JID), display_name
- **campaign_target_lists** — campaign_id, target_list_id (M2M join)
- **dispatches** — id, campaign_id (FK), post_id (FK), target_id (FK), wave_number (int), scheduled_at, wa_poster_id (FK, nullable), resolved_variant_id (FK, nullable), resolved_caption (text), status (enum: pending, in_progress, done, sent, failed, delayed, missed), actual_sent_at, error_message, created_at
- **wa_posters** — id, organization_id, name, provider_type (enum: watsender, twilio, unofficial_api), api_key, base_url, session_id, status (enum: active, disconnected, error), created_at
- **contact_numbers** — id, organization_id, phone_number, country_code, label, is_also_wa_poster (bool), wa_poster_id (FK, nullable)
- **caption_templates** — id, organization_id, name, body (text with variable placeholders), created_at
- **prefilled_message_templates** — id, organization_id, name, body, created_at
- **discount_rules** — id, organization_id, name, type (enum: product, category, keyword, date_range, time_frame, import_batch), target_id (polymorphic — references different tables based on type), discount_type (enum: fixed, percentage), discount_value, priority, starts_at, ends_at, created_at
- **discount_rule_products** — discount_rule_id, product_id (M2M for product-specific rules)
- **anti_detection_configs** — id, campaign_id (FK, unique), max_posts_per_hour, min_delay_seconds, max_delay_seconds, randomization_jitter_seconds, wa_poster_rotation_count, product_target_cooldown_hours, variant_cycle_strategy (enum: sequential, random), randomize_target_order_per_wave (bool), ai_rewrite_per_recipient (bool), forward_grouping (bool)
- **action_logs** — id, organization_id, actor_id (FK → auth.users, nullable for system actions), action_code (text — language-agnostic code like 'dispatch_marked_done'), entity_type, entity_id, changes (JSONB), reason (text, nullable), ip_address, user_agent, metadata (JSONB), created_at
- **ai_configs** — id, organization_id, feature_name (enum: post_suggestion, message_rewrite, import_parsing), enabled (bool), provider (enum: vertex_ai, saas_key, byok), api_key (encrypted), model_name, token_limit, created_at
- **ai_usage_logs** — id, organization_id, feature_name, model_used, tokens_consumed, estimated_cost, created_at
- **notifications** — id, organization_id, user_id (FK → auth.users), channel (enum: in_app, push, email), title_code, body_code, payload (JSONB), read_at, created_at
- **human_poster_working_hours** — id, user_id (FK → auth.users), day_of_week (int 0-6), start_time, end_time
- **human_poster_assignments** — campaign_id (FK), user_id (FK → auth.users) (M2M)

## Requirements

1. Every table has `organization_id` (except organizations and auth-referencing tables at top level)
2. Every table has RLS enabled (policies defined in T-009)
3. All timestamps use `timestamptz`
4. All IDs use `uuid` with `gen_random_uuid()` default
5. Extensions enabled: `pgcrypto`, `ltree`, `pg_cron`, `pg_partman`
6. Composite indexes for hot query paths (see dispatch queries, product search, action log retrieval)
7. Partitioning setup for `dispatches` (range on `scheduled_at`, monthly) and `action_logs` (range on `created_at`, monthly)
8. Foreign keys with appropriate ON DELETE behavior (CASCADE where ownership is clear, SET NULL where preservation matters)
9. JSONB fields documented with expected shape in comments
10. Enum types defined at schema level before table creation
