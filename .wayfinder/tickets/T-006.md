# T-006: API Route Contract ‚Äî WhatsApp Routes (/api/wa/*)

`wayfinder:task` | Parent: map.md

## Question

Produce the complete API route specification for all WhatsApp automation REST endpoints. These routes absorb WassFlow's existing API surface but add organization scoping and redesigned request/response shapes consistent with the wa schema.

## Scope

Must cover these route groups:

### Instances (WhatsApp Sessions)
- `GET /api/wa/instances` ‚Äî list org's WhatsApp instances
- `POST /api/wa/instances` ‚Äî register new instance
- `GET /api/wa/instances/:id` ‚Äî instance details with status
- `PATCH /api/wa/instances/:id` ‚Äî update config
- `DELETE /api/wa/instances/:id` ‚Äî remove instance
- `POST /api/wa/instances/:id/connect` ‚Äî initiate pairing/QR flow
- `GET /api/wa/instances/:id/qr` ‚Äî get current QR code
- `POST /api/wa/instances/:id/sync` ‚Äî sync groups and contacts

### Sequences (Automation Workflows)
- `GET /api/wa/sequences` ‚Äî list
- `POST /api/wa/sequences` ‚Äî create
- `GET /api/wa/sequences/:id` ‚Äî with steps
- `PATCH /api/wa/sequences/:id` ‚Äî update
- `DELETE /api/wa/sequences/:id` ‚Äî delete
- `POST /api/wa/sequences/:id/steps` ‚Äî add step
- `PATCH /api/wa/sequences/:id/steps/:stepId` ‚Äî update step
- `DELETE /api/wa/sequences/:id/steps/:stepId` ‚Äî remove step
- `PATCH /api/wa/sequences/:id/steps/reorder` ‚Äî reorder steps

### Triggers (Keyword ‚Üí Sequence)
- `GET /api/wa/triggers` ‚Äî list
- `POST /api/wa/triggers` ‚Äî create
- `PATCH /api/wa/triggers/:id` ‚Äî update
- `DELETE /api/wa/triggers/:id` ‚Äî delete

### Inbox Messages
- `GET /api/wa/inbox` ‚Äî list messages (filterable by instance, direction, status, keyword, date range)
- `GET /api/wa/inbox/:id` ‚Äî single message details
- `POST /api/wa/inbox/:id/reply` ‚Äî send reply

### WhatsApp Campaigns (Bulk Send)
- `GET /api/wa/campaigns` ‚Äî list
- `POST /api/wa/campaigns` ‚Äî create
- `GET /api/wa/campaigns/:id` ‚Äî details with events
- `PATCH /api/wa/campaigns/:id` ‚Äî update
- `DELETE /api/wa/campaigns/:id` ‚Äî delete
- `POST /api/wa/campaigns/:id/start` ‚Äî start sending
- `POST /api/wa/campaigns/:id/pause` ‚Äî pause
- `POST /api/wa/campaigns/:id/resume` ‚Äî resume
- `POST /api/wa/campaigns/:id/cancel` ‚Äî cancel
- `GET /api/wa/campaigns/:id/events` ‚Äî list campaign events (paginated, with status filter)
- `POST /api/wa/campaigns/:id/rerun-failed` ‚Äî retry failed events
- `POST /api/wa/campaigns/:id/control` ‚Äî engine control (tick trigger)

### Group Lists
- `GET /api/wa/group-lists` ‚Äî list
- `POST /api/wa/group-lists` ‚Äî create
- `GET /api/wa/group-lists/:id` ‚Äî with items
- `PATCH /api/wa/group-lists/:id` ‚Äî update
- `DELETE /api/wa/group-lists/:id` ‚Äî delete
- `POST /api/wa/group-lists/:id/items` ‚Äî add group(s)
- `DELETE /api/wa/group-lists/:id/items/:itemId` ‚Äî remove group

### Send Queue
- `GET /api/wa/send-queue` ‚Äî list (filterable by status, session, date range)
- `GET /api/wa/send-queue/:id` ‚Äî single item details

### Webhooks
- `POST /api/wa/webhooks/whatsapp/:session_id` ‚Äî incoming WhatsApp message webhook
- `POST /api/wa/webhooks/status` ‚Äî delivery status callback from WatsSender

### Engine
- `POST /api/wa/engine/tick` ‚Äî trigger engine processing cycle
- `GET /api/wa/engine/status` ‚Äî engine health and queue depth
