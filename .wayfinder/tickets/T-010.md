# T-010: AI Integration Specification

`wayfinder:task` | Parent: map.md

## Question

Produce the specification for AI features: Post ordering suggestions, message rewriting for individual targets, import parsing fallback, and the AI learning loop from Action Log data.

## Scope

1. **AI service interface** — abstract interface for AI providers (Vertex AI, global SaaS key, BYOK)
2. **AI Post Suggestion** — input (campaign context, product list, category data), output (suggested ordering array), adaptation mechanism (user reorder → AI adjusts immediately within campaign)
3. **AI Message Rewrite** — input (original caption, target info, past rewrites for same target), output (rewritten caption), uniqueness constraint (different from previous rewrites)
4. **AI Import Parsing** — input (Extract text + media context), output (suggested name, price, description), confidence score, fallback chain (Parsing Rules → AI → manual)
5. **Learning loop** — how Action Log data is used for fine-tuning or prompt improvement, data preparation format, training trigger (manual? scheduled? continuous?)
6. **Token tracking** — consumption logging per feature per organization, cost estimation, budget alerts
7. **Prompt engineering** — system prompts per feature, few-shot examples, output format constraints (JSON schema enforcement)
8. **Error handling** — AI timeout, hallucination detection, rate limiting, cost overrun prevention

## Assets produced

- `specs/ai/ai-service-interface.ts` — TypeScript interface
- `specs/ai/post-suggestion-spec.md` — AI Post Suggestion feature spec with prompts
- `specs/ai/message-rewrite-spec.md` — AI Message Rewrite feature spec with prompts
- `specs/ai/import-parsing-spec.md` — AI Import Parsing fallback spec with prompts
- `specs/ai/learning-loop-spec.md` — Action Log → AI training data pipeline
- `specs/ai/token-tracking-spec.md` — cost monitoring and budget enforcement
